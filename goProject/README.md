# Go 语言开发规范
##基础规范类
package规范
1. package命名时应采用有意义的包名，不能与标准库冲突。
2. 我们在包引用的时候，禁止使用相对路径，应采用go module方式进行引用。
```cassandraql
//错误方式
import "../goDemo"

//正确方式
import "github.com/goDemo"

```
3.变量与常量命名采用驼峰标准，不能使用_。
4.行尾不添加分号，该操作由编译时自动完成。
5.每行语句单独一行，禁止多行语句放在同一行。
6.每个函数命名时注意首字母大小写，只有该函数是public时，方可使用大写开头。
